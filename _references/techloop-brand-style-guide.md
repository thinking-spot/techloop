## Visual Identity

### Logo System

#### Primary Logo: Lowercase Wordmark
```
techloop
```

**Specifications:**
- Typography: Helvetica Neue Bold (or Helvetica Neue Medium for lighter applications)
- Case: Always lowercase, single word
- Special feature: Subtle infinity symbol integrated into "oo" of "loop"
- Letter spacing: -0.02em (slight tightening)
- Never: TechLoop, Tech Loop, TECHLOOP, Techloop

**Logo Variations:**

1. **Primary Wordmark** ‚Äî Full "techloop" wordmark
   - Use: Website header, email signatures, marketing materials
   - Minimum width: 120px
   - Clear space: Minimum 1x the height of lowercase "t" on all sides

2. **Icon Mark** ‚Äî Standalone loop/infinity symbol
   - Use: App icon, favicon, social media profile pictures
   - Dimensions: Perfect square (1:1 ratio)
   - Contains: Simplified infinity loop from "oo" design

3. **Lockup** ‚Äî Icon + Wordmark (horizontal)
   - Use: Branded merchandise, partnership materials
   - Ratio: Icon height = 1x wordmark "t" height
   - Spacing: 0.5x "t" height between icon and wordmark

**Logo Colors:**
- Primary: Headline (#094067) on white/light backgrounds
- Reversed: White (#FFFFFE) on dark backgrounds
- Accent: Button Blue (#3DA9FC) for special applications only
- Never use: Paragraph gray, Secondary background as logo color

**Logo Don'ts:**
- ‚ùå Don't stretch or distort proportions
- ‚ùå Don't add effects (shadows, gradients, outlines)
- ‚ùå Don't change typeface
- ‚ùå Don't rearrange elements
- ‚ùå Don't use on busy backgrounds without clear space
- ‚ùå Don't use all-caps or title case

## Color System

### Primary Palette

**Background (White)**
- Hex: `#FFFFFF`

- Use: Primary background, cards, modals
- Accessibility: WCAG AAA with all text colors

**Headline (Deep Navy)**
- Hex: `#094067`
- RGB: `9, 64, 103`
- Use: Headlines, primary text, important UI elements
- Accessibility: AAA on white background (12.74:1 ratio)

- Hex: `#5F6C7B`
- RGB: `95, 108, 123`
- Use: Body text, descriptions, secondary information
- Accessibility: AAA on white for body text (7.23:1 ratio)

- Hex: `#3DA9FC`
- RGB: `61, 169, 252`
- Use: Primary CTAs, active states, links, accents
- Pair with: Button Text (#FFFFFE)

**Button Text (Pure White)**
- Hex: `#FFFFFE`
- RGB: `255, 255, 254`
- Use: Text on button blue, reversed text on dark backgrounds
- Note: Near-white (#FFFFFE) prevents pure white rendering issues

**Secondary Background (Sky Blue)**
- Hex: `#90B4CE`
- RGB: `144, 180, 206`
- Use: Section backgrounds, subtle dividers, inactive states
- Accessibility: 2.89:1 with white (use for decorative only, not text backgrounds)

- Use: Error messages, destructive actions, critical alerts

### Color Usage Guidelines

**Backgrounds:**
- Primary surface: #FFFFFF
- Subtle sections: #90B4CE (use sparingly)
- Never use Button Blue (#3DA9FC) as large background

- H1-H2: #094067 (Headline)
- H3-H6: #094067 or #5F6C7B depending on emphasis

**Interactive Elements:**
- Primary CTA: #3DA9FC background + #FFFFFE text
- Secondary CTA: #FFFFFE background + #094067 text + #094067 border
- Tertiary CTA: Transparent background + #3DA9FC text
- Hover: Darken primary color by 10%
- Active: Darken primary color by 15%
- Focus: 2px outline in #3DA9FC

## Typography

**Display (Hero Headlines)**
- Font: Helvetica Neue Bold

- Line Height: 1.1 (61.6px)
- Letter Spacing: -0.02em

- Color: #094067

- Font: Helvetica Neue Bold

- Line Height: 1.2 (48px)
- Letter Spacing: -0.01em
- Use: Page headlines, primary content titles
- Color: #094067

- Font: Helvetica Neue Medium
- Size: 32px / 2rem
- Line Height: 1.25 (40px)
- Letter Spacing: -0.01em
- Use: Major sections, card headers
- Color: #094067

- Font: Helvetica Neue Medium

- Font: Inter Regular (400)

- Font: Inter Regular (400)

**Body Small**
- Font: Inter Regular (400)
- Size: 14px / 0.875rem

**Label**
- Font: Inter Medium (500)
- Size: 14px / 0.875rem
- Line Height: 1.4 (19.6px)
- Letter Spacing: 0.01em
- Text Transform: Uppercase
- Use: Form labels, tags, categories
- Color: #5F6C7B

**Button Text**
- Font: Inter Semibold (600)
- Size: 16px / 1rem
- Line Height: 1.5 (24px)
- Letter Spacing: 0.01em
- Use: Buttons, CTAs
- Color: #FFFFFE on buttons, #3DA9FC for text-only

**Caption**
- Font: Inter Regular (400)
- Size: 12px / 0.75rem
- Line Height: 1.4 (16.8px)
- Letter Spacing: 0
- Use: Fine print, timestamps, metadata
- Color: #5F6C7B at 80% opacity

### Typography Pairings

**Helvetica Neue Usage:**
- Brand moments (logo, taglines)
- Major headlines (Display, H1, H2)
- Marketing hero sections
- Emphasis and impact

**Inter Usage:**
- UI elements and interface text
- Body copy and reading content
- Forms, labels, and inputs
- Navigation and menus
- Data and numbers

**Pairing Principle:**
Helvetica Neue commands attention and communicates brand.
Inter ensures readability and usability.

### Typography Guidelines

**Line Length:**
- Optimal: 60-75 characters per line
- Maximum: 90 characters per line
- Use max-width: 65ch for body text

**Paragraph Spacing:**
- Between paragraphs: 1em (match font size)
- After headers: 0.5em
- Before new sections: 2em

**Alignment:**
- Body text: Left-aligned (never justified)
- Headlines: Left-aligned or center (context dependent)
- Buttons: Center-aligned text

**Responsive Type Scale:**
```css
/* Mobile (<640px) */
Display: 36px
H1: 32px
H2: 24px
Body: 16px

/* Tablet (641-1024px) */
Display: 48px
H1: 36px
H2: 28px
Body: 16px

/* Desktop (>1024px) */
Display: 56px
H1: 40px
H2: 32px
Body: 16px
```

## Layout & Spacing

### Grid System

**Container:**
- Max-width: 1280px
- Padding: 24px (mobile), 40px (tablet), 80px (desktop)
- Centered with `margin: 0 auto`

**Columns:**
- 12-column grid
- Gutter: 24px
- Responsive breakpoints:
  - Mobile: 1 column
  - Tablet: 6 or 4 columns
  - Desktop: 12, 6, 4, or 3 columns

### Spacing Scale

Use 8px base unit for consistency:

```
4px   ‚Äî xs  (tight spacing, form elements)
8px   ‚Äî sm  (compact UI)
12px  ‚Äî md  (default spacing)
16px  ‚Äî lg  (comfortable spacing)
24px  ‚Äî xl  (section spacing)
32px  ‚Äî 2xl (major sections)
48px  ‚Äî 3xl (page sections)
64px  ‚Äî 4xl (hero sections)
96px  ‚Äî 5xl (major dividers)
```

**Component Spacing:**
- Between related elements: 8-12px
- Between component groups: 24px
- Between page sections: 48-64px
- Top/bottom page padding: 64-96px

## Imagery & Photography

### Photography Style

**Lifestyle Photography:**
- Real people using AI wearables in authentic contexts
- Natural lighting, not overly staged
- Focus on the experience, not just the device
- Diverse representation (age, ethnicity, gender, ability)
- Locations: Urban environments, modern workspaces, outdoor activities
- Mood: Confident, forward-looking, accessible

**Product Photography:**
- Clean, Apple-style product shots
- White or subtle gradient backgrounds
- Multiple angles showing device details
- Consistent lighting and shadows
- High resolution (min 2000px width)

**Photography Composition:**
- Rule of thirds
- Lots of negative space
- Subject in focus, background slightly blurred
- Avoid clutter and busy backgrounds

### Image Treatment

**Overlay for Text Legibility:**
```css
background: linear-gradient(
  135deg,
  rgba(9, 64, 103, 0.7) 0%,
  rgba(61, 169, 252, 0.3) 100%
)
```

**Image Ratios:**
- Hero banners: 21:9 or 16:9
- Device cards: 4:3
- Thumbnails: 1:1 (square)
- Full-width sections: 16:9 or 3:1

### Illustration Style

**When to use illustrations:**
- Explaining abstract concepts (how rental works)
- Empty states (no devices yet)
- Error states (404, connection issues)
- Onboarding flows

**Style characteristics:**
- Flat, geometric shapes
- Minimal detail
- Color palette: Main (#FFFFFE), Highlight (#3DA9FC), Secondary (#90B4CE)
- Stroke weight: 2-3px in #094067
- Rounded corners (8-12px radius)
- Subtle shadows (0 2px 8px rgba(9, 64, 103, 0.1))

## Motion & Animation

### Animation Principles

**Purpose:** Enhance usability, provide feedback, create delight

**Duration:**
- Micro-interactions: 150-200ms
- Component transitions: 250-300ms
- Page transitions: 400-500ms
- Never exceed 600ms

**Easing:**
- Default: `ease-out` (decelerating)
- Entrances: `ease-out`
- Exits: `ease-in`
- Two-way: `ease-in-out`
- Bouncy effects: `cubic-bezier(0.68, -0.55, 0.265, 1.55)`

### Common Animations

**Button Hover:**
```css
transition: all 0.2s ease-out;
transform: translateY(-1px);
box-shadow: 0 4px 12px rgba(61, 169, 252, 0.2);
```

**Card Hover:**
```css
transition: all 0.3s ease-out;
transform: translateY(-4px);
box-shadow: 0 8px 24px rgba(0, 0, 0, 0.12);
```

**Modal Enter:**
```css
@keyframes modalEnter {
  from {
    opacity: 0;
    transform: scale(0.95) translateY(20px);
  }
  to {
    opacity: 1;
    transform: scale(1) translateY(0);
  }
}
animation: modalEnter 0.3s ease-out;
```

**Fade In:**
```css
@keyframes fadeIn {
  from { opacity: 0; }
  to { opacity: 1; }
}
animation: fadeIn 0.4s ease-out;
```

**Slide In (from right):**
```css
@keyframes slideInRight {
  from {
    opacity: 0;
    transform: translateX(30px);
  }
  to {
    opacity: 1;
    transform: translateX(0);
  }
}
animation: slideInRight 0.4s ease-out;
```

**Loading Spinner:**
```css
@keyframes spin {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}
animation: spin 1s linear infinite;
```

### Interaction Feedback

**Click/Tap:**
- Scale down 2-3%: `transform: scale(0.98)`
- Duration: 100ms
- Return to normal: 150ms

**Focus States:**
- Always visible (accessibility)
- 2px outline in #3DA9FC
- 2px offset from element
- Never remove focus indicators

**Loading States:**
- Skeleton screens (gray pulse animation)
- Spinners for short waits (<3 seconds)
- Progress bars for longer operations
- Never block entire page unless necessary

## Voice & Tone

### Brand Voice

**Core Attributes:**
1. **Clear** ‚Äî No jargon, explain tech simply
2. **Confident** ‚Äî We know our stuff, we're the experts
3. **Helpful** ‚Äî Always solving user problems
4. **Forward-looking** ‚Äî Tomorrow's tech, accessible today

### Tone by Context

**Marketing Pages:**
- Tone: Aspirational but approachable
- Example: "Try tomorrow's tech, today. No commitment, just curiosity."
- Avoid: Over-promising, hype, tech jargon

**Product Descriptions:**
- Tone: Informative and practical
- Example: "Meta Ray-Ban glasses let you capture moments hands-free and stay connected with AI assistance."
- Avoid: Marketing fluff, unnecessary adjectives

**Transactional Emails:**
- Tone: Professional and reassuring
- Example: "Your XREAL Air 2 Pro is on its way. Expected arrival: Dec 15."
- Avoid: Overly casual, emojis, exclamation points

**Error Messages:**
- Tone: Empathetic and solution-focused
- Example: "We couldn't process your payment. Please check your card details or try a different payment method."
- Avoid: Blaming user, technical error codes without explanation

**Help/Support:**
- Tone: Patient and thorough
- Example: "Need help getting started? We're here every step of the way."
- Avoid: Condescension, assuming user knowledge

### Writing Guidelines

**Headlines:**
- Short and punchy (5-10 words)
- Lead with benefit, not feature
- Use active voice
- Example: "Rent AI wearables. Return anytime."

**Body Copy:**
- Scannable (short paragraphs, bullet points)
- One idea per paragraph
- Use second person ("you," not "users")
- Explain technical terms in plain language

**CTAs:**
- Action-oriented verbs
- Create urgency without pressure
- Be specific about what happens next
- Good: "Start Your Trial" (clear outcome)
- Bad: "Click Here" (vague)

**Product Naming:**
- Use manufacturer's exact product names
- Example: "Meta Ray-Ban Smart Glasses" (not "Meta Glasses")
- Category in sentence case: "smart glasses," "smart ring"
- Company name: Always "techloop" (lowercase)

### Word Choice

**Use:**
- Rent, rental, subscription
- Try, test, experience
- Commitment-free, flexible, risk-free
- Latest, cutting-edge, tomorrow's tech
- Affordable, accessible, premium

**Avoid:**
- Lease, borrow (legal implications)
- Revolutionary, game-changing (hyperbole)
- Best, perfect, ultimate (unverifiable claims)
- Cheap, bargain (diminishes premium positioning)
- Disrupt, innovate (overused startup jargon)

## Web Experience: "Already In" Design

### Concept

Users should feel like they're **already using the product** from the moment they land on the marketing site. The marketing website is built within the web app UX and exposed to non-logged-in users.

This creates:
- **Immediate familiarity** ‚Äî "I can see how this works"
- **Reduced friction** ‚Äî No transition shock from marketing to app
- **Trust** ‚Äî Transparency about the actual experience
- **Conversion** ‚Äî Users can imagine themselves using it

### Implementation

## Accessibility

### Color Contrast

All text meets WCAG AA standards minimum:
- **AAA:** Headline (#094067) on white ‚Äî 12.74:1
- **AAA:** Paragraph (#5F6C7B) on white ‚Äî 7.23:1
- **AA Large:** Button (#3DA9FC) on button text (#FFFFFE) ‚Äî 3.12:1

**Never use:**
- Secondary Background (#90B4CE) as text background
- Light text on light backgrounds
- Color as only indicator (use icons + text)

### Keyboard Navigation

**All interactive elements must:**
- Be keyboard accessible (Tab, Enter, Space)
- Show clear focus indicators
- Follow logical tab order
- Support Escape to close modals

**Focus Indicators:**
```css
:focus-visible {
  outline: 2px solid #3DA9FC;
  outline-offset: 2px;
}
```

### Screen Readers

**Images:**
- All `<img>` tags need descriptive `alt` text
- Decorative images: `alt=""`
- Complex images: Provide longer description

**Buttons:**
- Use `<button>` element (not `<div>` with onclick)
- Include text label (not icon-only)
- Or use `aria-label` for icon buttons

**Forms:**
- Every input has associated `<label>`
- Error messages announced to screen readers
- Use `aria-describedby` for helper text

**ARIA Landmarks:**
```html
<header role="banner">
<nav role="navigation">
<main role="main">
<aside role="complementary">
<footer role="contentinfo">
```

### Motion Sensitivity

Respect `prefers-reduced-motion`:
```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

- Works perfectly with Tailwind

**Key components to build first:**
1. Button (all variants)
2. Input, Select, Checkbox
3. Card
4. Modal/Dialog
5. Navigation (top nav + sidebar)
6. Device Card (custom component)

### Performance Guidelines

**Images:**
- Use Next.js Image component (`<Image />`)
- Serve WebP with fallback
- Lazy load below-fold images
- Responsive images (srcset)

**Fonts:**
- Self-host fonts (don't use Google Fonts CDN)
- Use `font-display: swap`
- Subset fonts to needed characters
- Preload critical fonts

**CSS:**
- PurgeCSS with Tailwind (remove unused)
- Critical CSS inline
- Defer non-critical CSS

**JavaScript:**
- Code splitting (Next.js automatic)
- Lazy load components not in viewport
- Tree-shaking enabled

**Target Performance:**
- Lighthouse Score: 90+ (all categories)
- First Contentful Paint: <1.5s
- Time to Interactive: <3.5s
- Cumulative Layout Shift: <0.1
